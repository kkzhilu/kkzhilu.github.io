@ModelAttribute注解

Controller层一般都单独作为一个类使用，但是一个合理的设计，可以设计如 BaseController，通过继承的方式，让所有的controller都可以有一定的公共参数

// 该注解会在所有方法之前被调用，可以用作公共配置的设置等等
@ModelAttribute
public void testModelAttribute (HttpServletRequest request) {
	System.out.println("1111");
	request.setAttribute("myTest", "my test");
}

// 加次注解后，@RequestMapping失效，请求会报404错误，但是方法仍会依照ModelAttribute特性执行（先执行）
@ModelAttribute 
@RequestMapping(value = "/users", method = RequestMethod.GET)
public String restful (HttpServletRequest request) {
	System.out.println("查询所有");
	System.out.println(request.getAttribute("myTest"));
	return "ok";
}

统一异常处理 - 接口实现

详情见：KerwinBoots - boot_code_generator分支

@Configuration
public class WebConfigurer implements WebMvcConfigurer {

    /***
     * 统一异常处理
     */
    @Override
    public void configureHandlerExceptionResolvers(List<HandlerExceptionResolver> exceptionResolvers) {
        exceptionResolvers.add((request, response, handler, e) -> {
            Result result = new Result();

            //业务失败的异常，如[账号或密码错误]
            if (e instanceof ServiceException) {
                result.setCode(ResultCode.FAIL).setMessage(e.getMessage());
                logger.info(e.getMessage());

            } else if (e instanceof NoHandlerFoundException) {
                result.setCode(ResultCode.NOT_FOUND).setMessage("接口 [" + request.getRequestURI() + "] 不存在");

            } else if (e instanceof ServletException) {
                result.setCode(ResultCode.FAIL).setMessage(e.getMessage());

            } else {
                result.setCode(ResultCode.INTERNAL_SERVER_ERROR).setMessage("接口 [" + request.getRequestURI() + "] 内部错误，请联系管理员");
                String message;
                if (handler instanceof HandlerMethod) {
                    HandlerMethod handlerMethod = (HandlerMethod) handler;
                    message = String.format("接口 [%s] 出现异常，方法：%s.%s，异常摘要：%s",
                            request.getRequestURI(),
                            handlerMethod.getBean().getClass().getName(),
                            handlerMethod.getMethod().getName(),
                            e.getMessage());
                } else {
                    message = e.getMessage();
                }

                logger.error(message, e);
            }

            responseResult(response, result);
            return new ModelAndView();
        });
    }

    private void responseResult(HttpServletResponse response, Result result) {
        response.setCharacterEncoding("UTF-8");
        response.setHeader("Content-type", "application/json;charset=UTF-8");
        response.setStatus(200);
        try {
            response.getWriter().write(JSON.toJSONString(result));
        } catch (IOException ex) {
            logger.error(ex.getMessage());
        }
    }
}

@ResponseBody 注解作用及原理

MVC中@ResponseBody可以把结果返回值以JSON字符串的形式输出，所以@ResponseBody注解到底做了什么事？

  1. ModelAndView 类型对象会有JSPView处理器进行处理，而@ResponseBody会通过response进行输出
  2. HttpMessageConverter 进行数据转换，把输入流转化成对象，对象返回值转化为输出流，而@ResponseBody注解工作的实质即以下两行代码
response.setContentType("application/json;charset=UTF-8");
response.getWriter().write(JSON.toJSONString(invoke));

无论是调用该注解，或者自行response输出json字符串，返回结果都可以保证是JSON字符串，但为什么只有加了注解的才可以在ajax中，直接作为对象使用？ 关键在于头信息

JQuery有内部处理机制去识别 -》 application/json;charset=UTF-8

关键词：【HttpMessageConverter】

参考博文： https://www.jianshu.com/p/333ed5ee958d

@Service，@Controller注解互换可以吗

@Service，@Controller的本质和@Component一致，都是为了标明一个bean

因此，@Service注解换成@Controller，或者@Component都是可以的，@Service只是用作业务上的区分

@Controller不同，它还包含@RequestMapping注解的配合使用，如果用@service代替，会让所有的mapping都失效，所有请求都是404
